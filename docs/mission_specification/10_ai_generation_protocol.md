# Phoenix Mission Specification — Part 10: AI Generation Protocol

**Specification Version:** 1.0.0  

---

## 1. Purpose

This protocol enables AI systems to generate conformant mission YAML files from structured prompts. It ensures consistency across 500+ missions while maintaining the quality standards defined in Parts 01–09.

AI generation is a **draft production tool**. Every AI-generated mission STILL requires human review before production deployment.

---

## 2. AI Generation Prompt Template

When requesting a new mission from an AI, provide this structured prompt:

```
SYSTEM: You are a professional German language curriculum designer 
specializing in call center and customer service training for 
Moroccan Arabic (Darija) speakers. You generate mission files 
conforming to the Phoenix Mission Specification v1.0.0.

All German must be grammatically perfect (verify against Duden standards).
All Darija must be natural Moroccan Arabic (not MSA).
All phrases must be chunks (not isolated words).
Speaking exercises must comprise ≥60% of exercise time.

GENERATE A PHOENIX MISSION:

Mission Parameters:
  Module: {1-12}
  Sequence: {1-15}
  Type: {discovery | practice | challenge | simulation | recovery}
  CEFR Level: {A0 | A1 | A2 | B1}
  Difficulty: {1-10}
  Career Domain: {from controlled vocabulary}
  Theme: "{one-sentence description of what the learner DOES}"
  Character: {character_id from registry}
  Character Mood: {from mood vocabulary}
  
New Vocabulary (3-5 chunks):
  1. "{German chunk}" = "{English meaning}"
  2. "{German chunk}" = "{English meaning}"
  3. "{German chunk}" = "{English meaning}"
  
Grammar Pattern (embedded, not stated):
  Pattern ID: {from grammar pattern registry}
  
Builds On (prerequisites):
  - {mission_id_1}
  - {mission_id_2}

Review Items (from previous missions):
  - {vocabulary_id_1}
  - {vocabulary_id_2}

OUTPUT: Complete YAML file conforming to PMS spec_version 1.0.0
including all sections: metadata, learning_objective, scenario,
emotional_goal, vocabulary, dialogue, exercises, grammar_discovery,
pronunciation_coaching, common_mistakes, cultural_notes, smart_review,
confidence_scoring, success_criteria, career_transfer, quality_report.
```

## 3. AI Generation Constraints

### 3.1 What AI CAN Do

| Capability | Conditions |
|-----------|-----------|
| Generate complete YAML structure | Following schema exactly |
| Write German dialogue and vocabulary | Must be verified post-generation |
| Create exercise sequences with scaffolding | Following phase assignment rules |
| Assign difficulty and CEFR levels | Based on content complexity |
| Generate pronunciation notes | For known phonetic challenges |
| Write feedback text (English + Darija) | Following approved tone |
| Generate cultural notes | Based on documented cultural contrasts |
| Write career transfer statements | Based on real workplace data |
| Set `grammar_confidence_percent: 95` | If using grammar verification tools |
| Set `cefr_alignment_verified: true` | If following official CEFR word lists |

### 3.2 What AI CANNOT Do

| Limitation | Reason | Consequence |
|-----------|--------|-------------|
| Generate audio files | Requires professional recording | `audio_recorded: false` always |
| Verify vocabulary naturalness | Requires native speaker intuition | `vocabulary_verified: false` |
| Confirm cultural accuracy | Requires lived German workplace experience | `cultural_accuracy_verified: false` |
| Verify Darija translations | Requires native Moroccan speaker | `darija_translation_verified: false` |
| Generate images | Requires separate image pipeline | Image paths left as placeholders |
| Set `quality_report` fields to true | Most require human verification | Honest reporting required |

### 3.3 AI Quality Report Template

AI-generated missions MUST include this honest quality report:

```yaml
quality_report:
  grammar_confidence_percent: 95               # High but not 99 (needs human check)
  vocabulary_verified: false                   # AI cannot verify naturalness
  idiomatic_expressions_reviewed: false        # Requires native speaker
  cefr_alignment_verified: true                # AI can verify against word lists
  audio_recorded: false                        # Always false for AI-generated
  cultural_accuracy_verified: false            # Requires human with German experience
  darija_translation_verified: false           # Requires Moroccan native speaker
  review_date: "{generation_date}"
  reviewer: "AI-generated (requires human review)"
  notes: "Generated by AI. German grammar self-verified. Requires native speaker review for vocabulary naturalness, cultural accuracy, and Darija translations."
```

## 4. Validation Pipeline

### 4.1 Automated Validation (Layer 1)

Run immediately after generation:

```
STRUCTURAL CHECKS:
  ✓ YAML parses without errors
  ✓ spec_version = "1.0.0"
  ✓ All required fields present and non-empty
  ✓ metadata.id follows naming convention (m{xx}_{xxx}_{type})
  ✓ metadata.skills sum = 100
  ✓ metadata.skills.speaking ≥ 50 (for discovery/practice/challenge)
  ✓ vocabulary.core length ≤ 5 (discovery) or ≤ 3 (practice)
  ✓ All vocabulary_ids in exercises reference defined vocabulary
  ✓ All prerequisite mission IDs are valid format
  ✓ confidence_scoring.dimensions weights sum to 1.0
  ✓ Exercises span all four phases (ignite, equip, challenge, land)
  ✓ No duplicate exercise IDs
  ✓ Dialogue turns have speaker and type
  ✓ All production turns have acceptable_responses
  ✓ All production turns have three scaffolding levels
  ✓ career_transfer.statement_en is non-empty
  ✓ learning_objective starts with "After this mission, the learner can"
  ✓ estimated_duration_minutes ≤ 15
  ✓ srs_config.context_variations has ≥ 3 entries per item

LINGUISTIC CHECKS (automated):
  ✓ No English words in text_de fields (except proper nouns)
  ✓ All text_de fields contain at least one German word
  ✓ IPA transcriptions use valid IPA symbols
  ✓ Register field uses controlled vocabulary (formal/informal/neutral/universal)
  ✓ Character IDs reference the official registry
  ✓ Mood IDs reference the controlled vocabulary
  ✓ Career domains reference the controlled vocabulary
  ✓ Exercise types reference the type registry
  ✓ Grammar pattern_id references the pattern registry
```

### 4.2 Pedagogical Review (Layer 2 — Human)

| Check | Reviewer |
|-------|----------|
| Learning objective is truly observable and singular | Content expert |
| Scenario passes "Real Test" (would happen in German workplace) | Content expert |
| Difficulty progression within exercises is correct | Content expert |
| Scaffolding degrades naturally across the mission | Content expert |
| Feedback language is safe (no shame, no "wrong") | Content expert |
| Grammar is truly embedded (not stated or tested in isolation) | Content expert |
| Cultural notes are accurate and non-judgmental | Content expert |
| Exercise mix provides variety (no 3 same types in a row) | Content expert |
| New vocabulary items are genuinely useful for the target career | Content expert |

### 4.3 Quality Verification (Layer 3 — Expert)

| Check | Reviewer | Tool |
|-------|----------|------|
| All German is grammatically correct | Native German speaker | Duden Online, DWDS |
| Vocabulary sounds natural (not textbook-ish) | Native German speaker | Intuition + Reverso Context |
| CEFR level is accurate | CEFR-trained assessor | Goethe Wortlisten |
| Cultural notes reflect current German workplace reality | German resident | Lived experience |
| Darija translations are natural Moroccan Arabic | Moroccan native speaker | Native intuition |
| French bridges are accurate | French speaker | Phonetic verification |
| Audio quality meets studio standards | Audio engineer | Technical metrics |

## 5. Batch Generation Protocol

For generating multiple missions efficiently:

### 5.1 Sequence

```
Step 1: PLAN
  - Generate mission manifest (all IDs, themes, vocabulary targets for the module)
  - Verify no vocabulary overlap/conflicts
  - Map grammar patterns to missions
  - Assign characters ensuring variety

Step 2: GENERATE (in sequence order)
  - Generate missions in their sequence order within the module
  - Each mission's review items reference ONLY previously generated missions
  - Vocabulary IDs are globally unique across the batch
  - Characters rotate (no character appears in more than 3 consecutive missions)

Step 3: CROSS-VALIDATE
  - Verify all vocabulary_ids referenced in exercises exist in definitions
  - Verify all prerequisite IDs reference existing missions in the batch or earlier modules
  - Verify narrative continuity (characters' ongoing situations are consistent)
  - Verify no exact duplicate phrases across missions (variety requirement)
  - Verify CEFR level progression is monotonically non-decreasing within module

Step 4: AUTOMATED VALIDATION
  - Run all Layer 1 checks on every file in the batch
  - Generate validation report

Step 5: HUMAN REVIEW QUEUE
  - Flag all files for Layer 2 + Layer 3 review
  - Prioritize by: module order (earlier modules reviewed first)
```

### 5.2 Module Generation Template

```
Module {N} Generation Brief:

Module Theme: "{Narrative chapter}"
CEFR Range: {entry} → {exit}
Total Missions: {10-15}
  Discovery: {6-9}
  Practice: {2-3}
  Challenge: {1-2}
  Simulation: {0-1}

Primary Career Domain: {domain}
Primary Characters: {2-3 character_ids}
Grammar Patterns Embedded: {2-3 pattern_ids}
New Vocabulary Target: {30-50 chunks total across module}

Mission List:
  m{N}_001_d: "{Theme sentence}"
  m{N}_002_d: "{Theme sentence}"
  m{N}_003_d: "{Theme sentence}"
  m{N}_004_p: "{Theme sentence}"
  m{N}_005_d: "{Theme sentence}"
  ...
```

## 6. Version Control for Generated Content

### 6.1 File Naming

```
content_source/missions/m{module}/{mission_id}.yaml
```

### 6.2 Git Workflow

```
1. AI generates mission → saved to feature branch
2. Automated validation runs (CI/CD)
3. If validation passes → PR created for human review
4. Human reviewer (Layer 2) approves or requests changes
5. Expert reviewer (Layer 3) verifies German/Darija/culture
6. Merge to main → mission enters content database build pipeline
7. Content database rebuilt → mission available in app
```

### 6.3 Change Tracking

When a mission is updated:
- `metadata.version` increments
- `quality_report.review_date` updates
- Previous version remains in git history
- SRS items linked to changed vocabulary are flagged (but not reset)
- Learners who already completed the mission are NOT affected

## 7. Domain-Specific Generation Guidance

### 7.1 Call Center Missions (Modules 4–8)

```
EMPHASIS: Phone communication formulas, customer handling, formal register
CHARACTERS: Frau Schmidt (easy), Herr Weber (difficult), Frau Bauer (confused)
VOCABULARY CLUSTERS: openers, bridges, empathy, actions, closers
SCENARIOS: Incoming calls, order status, complaints, billing, returns
GRAMMAR (embedded): Modal verbs, V2 order, separable verbs, Perfekt
REGISTER: Strictly formal (Sie/Ihnen always)
SPEED: From Module 6: time_pressure exercises introduced
```

### 7.2 Interview Preparation (Module 9)

```
EMPHASIS: Self-presentation, extended production (30-90 seconds), formal register
CHARACTERS: Herr Braun (interviewer), system (coaching voice)
VOCABULARY CLUSTERS: transitions, affirmers, actions (past tense)
SCENARIOS: Self-introduction, strengths/weaknesses, situational questions
GRAMMAR (embedded): Perfekt (past experience), Konjunktiv II (hypotheticals)
SPECIAL: Personalized content (learner's actual background encoded)
DURATION: Longer exercises (60-90s production windows)
```

### 7.3 Tourism & Logistics Missions (Module 7)

```
EMPHASIS: Product knowledge, shipping vocabulary, troubleshooting steps
CHARACTERS: Herr Klein (impatient business customer), Lisa (IT colleague)
VOCABULARY CLUSTERS: quantifiers, actions, confirmers
SCENARIOS: Delivery tracking, return processes, account management
GRAMMAR (embedded): Passive voice ("Ihre Bestellung wird bearbeitet")
REGISTER: Formal with technical precision
SPECIAL: Number/address dictation exercises prominent
```
